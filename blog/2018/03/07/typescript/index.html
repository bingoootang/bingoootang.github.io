# 复杂应用中的 TypeScript 实践

TypeScript 是一个可以编译到纯 JavaScript(下称 JS) 的有类型定义的 JS 超集。接触 TypeScript 也有了一段时间，个人感觉，相较于 JS 来说更适合于生命周期较长的复杂单页应用。所谓生命周期较长的应用，就是应用存在时间有一个较长的时间跨度，通常有若干个开发和维护过程。

# TypeScript 实践

在长周期复杂应用的开发维护过程中，相较于 JS 而言，TypeScript 主要有如下几点优势：

  • 代码的静态检查
  • IDE 智能提示
  • 面向对象增强

# 代码的静态检查

相信刚开始使用 TypeScript 的同学很大一部分都是冲着 TypeScript 的类型检查去的。TypeScript 的类型检查与传统的 ESLint 不同。ESLint 通过解析抽象语法树的方式来判断代码是否存在问题，更多的是从语法层面进行检查。而 TypeScript 有着完备的类型检查能力，除了语法检查以外，还可以完成类型检查、类型推断等。社区中流行的库都有 TypeScript 类型声明，加上我们自己业务代码完善的类型定义，整个代码库都可以保持一个很高的健康度。

# 低级错误

举个例子，在开发过程中，我们偶尔会敲错 API，比如少敲了一个字母，

str1.localCompare(str2)
1

上面这个例子，如果通过 JS 来写，需要等到在浏览器中实际执行的时候我们才能发现问题。如果通过 TypeScript 来写的话，在编译阶段我们就可以发现问题。如果使用 vscode 等对 TypeScript 支持较好的 IDE，那在代码编写过程中就可以发现。

# 类型推断

在类型有限的小型应用中我们很少遇到类型推断的问题，但是在一个复杂的应用中，可能有很多个模块，每个模块中又有很多类型。如果使用 JS 来开发的话，我们很难搞清楚某个类型上到底有哪些方法、数据，往往需要在调试过程中才能明确。由于 TypeScript 的类型系统，我们可以轻松的在编码阶段确定具体的类型，以及该类型上有哪些方法、数据。比如：

// a.js
export default function A(param) {
  return param.toUpperCase();
}

// b.js
export default function B(param) {
  // some logic
  A(param);
}

B(['lowercase']);

# IDE 智能提示

IDE 智能提示也得益于 TypeScript 的静态类型系统。比如 VSCode，在 VSCode 中开发代码时，我们甚至都不需要查看 API，直接通过智能提示就可以便捷的查看相应的 API。现在流行的库如 React 、Redux 、 Lodash 、 Antd 等等都有 TypeScript 类型声明，加上我们自己业务代码完善的类型定义，整个代码库仿佛就是一个活的 API 文档。同时，丰富的类型声明也是静态检查得以完成的基础。

# 面向对象增强

虽然新的 JS 的规范引入了诸如 class 等概念来完善 JS 的类型系统，但是对于复杂的工程实践来说还远远不够。TypeScript 提供了完善的类型系统定义能力，以完成复杂的面向对象设计。

# 泛型

如果我们有一段比较通过的逻辑，但是参数类型是动态改变，那么泛型是比较好的模式。

class GenericCompare<T> {
  compare: (x: T, y: T) => boolean;
}
class NewType {
  value: string;
  constructor(value: string) {
    this.value = value;
  }
}
const instance = new GenericCompare<NewType>();
instance.compare = (x: NewType, y: NewType) => {
  return x.value.localeCompare(y.value) > 0;
};
const x = new NewType('x');
const y = new NewType('y');
instance.compare(x, y);

# 访问权限控制

JS 面向对象编程的一个大问题就是没有提供原生支持信息隐藏的方案，这一直是一个槽点。TypeScript 弥补了 JS 的不足，实现复杂系统中的信息隐藏。

class Person {
  public name: string;
  protected age: number;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

class Player extends Person {
  static someAttr = 1;
  public num: string;

  constructor(name: string, num: number, age: number) {
    super(name, age);
    this.num = num;
  }
}
let player = new Player("Player1", 11, 20);
console.log(player.name);

# 接口

通过组合继承类和实现接口的方式可以使面向对象编程更为灵活、可扩展性更好。

interface Machine {
  move(): void
}
interface Human {
  run(): void
}
class Base {
}
class Robot extends Base implements Machine, Human {
  run() {
    console.log('run');
  }
  move() {
    console.log('move');
  }
}

# 总结

现在的 Web 应用很多都是复杂的单页应用，需要一个团队来开发和维护。那么协作效率、代码可读性、可维护性、健壮性等等都是我们应该重点专注的。如果有有完善的类型定义，再加上智能的 IDE（如 VS Code），可以显著的提升代码的可读性、可维护性。很多时候都是看看源代码就能较容易的发现一些问题和 Bug。一些常用的第三方库和框架都有完善的 TypeScript 类型定义，同时整个项目业务代码都有完善的 TypeScript 类型定义，静态检查出错的都会在 IDE 中提示出来，甚至可以阻断构建流程，这样可以减少 Bug 的产生，代码质量更为可控，代码健康度更高，这在重构代码的时候特别有用。
