# 聊聊网页布局的那些事（二）

上一篇文章中讨论了关于网页适配的基本概念，主要集中在 viewport 的理解上。本篇主要讨论关于移动端网页适配的具体实践。

# 基于 rem 的适配方案

rem（font size of the root element），简单的理解，就是相对于根元素 <html> 的 font-size 来做计算。使用rem单位，能轻易的根据 <html> 的 font-size 计算出元素的盒模型大小。
有了 rem，我们可以轻松地做到不同手机屏幕的适配了。方法就是通过 window.innerWidth 来获取视觉视口的宽度，设置相应的 rem，然后网页中元素的长度单位都以 rem 来表示，即可做到不同屏幕的自适应了。
通常的做法是：

document.documentElement.style.fontSize = window.innerWidth / 10;

之所以要除以 10，是为了向后面说的 vw 方案靠拢，实际上可以除以任何值，或者干脆不除，只要保证元素的尺寸成比例的换算为设计稿上的尺寸就可以了。

# 淘宝 lib-flexible 方案

上面基于 rem 的适配比较原始，且未充分考虑到不同 dpr 的情况，淘宝的同学根据实践，总结出了 lib-flexible 方案。该方案的基本原理是，根据 dpr 动态设置网页的 meta viewport。

if (!dpr && !scale) {
  var isAndroid = win.navigator.appVersion.match(/android/gi);
  var isIPhone = win.navigator.appVersion.match(/iphone/gi);
  var devicePixelRatio = win.devicePixelRatio;
  if (isIPhone) {
    // iOS下，对于2和3的屏，用2倍的方案，其余的用1倍方案
    if (devicePixelRatio >= 3 && (!dpr || dpr >= 3)) {
      dpr = 3;
    } else if (devicePixelRatio >= 2 && (!dpr || dpr >= 2)) {
      dpr = 2;
    } else {
      dpr = 1;
    }
  } else {
    // 其他设备下，仍旧使用1倍的方案
    dpr = 1;
  }
  scale = 1 / dpr;
}

if (!metaEl) {
  metaEl = doc.createElement('meta');
  metaEl.setAttribute('name', 'viewport');
  metaEl.setAttribute('content', 'initial-scale=' + scale + ', maximum-scale=' + scale + ', minimum-scale=' + scale + ', user-scalable=no');
  if (docEl.firstElementChild) {
    docEl.firstElementChild.appendChild(metaEl);
  } else {
    var wrap = doc.createElement('div');
    wrap.appendChild(metaEl);
    doc.write(wrap.innerHTML);
  }
}

function refreshRem() {
  var width = docEl.getBoundingClientRect().width;
  if (width / dpr > 540) {
    width = 540 * dpr;
  }
  var rem = width / 10;
  docEl.style.fontSize = rem + 'px';
  flexible.rem = win.rem = rem;
}

上面这段代码是 lib-flexible 源码中的一部分，我们可以看到一个大概的逻辑。
在默认情况下，先获取布局视口的宽度，判断是否为移动设备（除以像素比得到设备宽度，参考讨论），之后，将 rem 设置为布局视口宽度的十分之一。设置缩放比例为像素比的倒数，从而将视觉视口的宽度设置为与布局视口的宽度相等，以完成页面完全展示。

关于 lib-flexible 的具体工程实践，可以参考 使用Flexible实现手淘H5页面的终端适配，这里不再赘述。
没有一个方案是完美的，lib-flexible 也有不足的地方，比如安卓机没有做适配，无法与响应式共存等。网上也有一些解决方法，比如 基于淘宝弹性布局方案lib-flexible的问题研究。

# 基于 vw 的适配方案

vw 是 viewport's width 的简写（这个视口指的是视觉视口），1vw 等于 window.innerWidth 的 1%。因此，基于 vw 的适配方案是由浏览器原生支持的，不需要再通过类似于 lib-flexible 通过 javascript 来模拟了。
下图是 vw 的相关定义：

由 vw 的定义以及视觉视口与理想视口的关系我们知道，在固定缩放比例的情况下，vw 只跟理想视口的尺寸有关，而理想视口的尺寸是由厂商自己决定的。因此，只要我们将设计稿中的尺寸等比例映射成 vw 表示的尺寸，就可以实现在不同设备上的适配了。

具体的工程实践可以参考再聊移动端页面的适配中的相关讨论，或者如何在Vue项目中使用vw实现移动端适配 中的相关实践，此处不再赘述。
