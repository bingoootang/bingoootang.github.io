# 打造高效的个人信息流

身处一个高度互联互通的时代，获取信息的成本很低，我们可以通过几乎免费的方式了解到最新的新闻、最新的技术等各种信息。但是信息不等于能力，只有能力才能助力个人成长。信息的互联互通同时也意味着信息的高度泛滥，每天大量的信息如洪水般冲击着我们的感官，争夺我们有限的注意力。无用信息占据的注意力越多，能力的形成速度越慢，成长越慢。因此如何从信息洪流中提取有效信息，加速从信息到能力的转化过程，是一个亟待的问题。

# 信息流

根据个人的学习经历，能力的形成过程可以概括为如下过程。

 学习 -> 使用 -> 学习 -> ... -> 学习 -> 使用 -> ... -> 使用 -> ... -> 总结 -> 经验 -> 能力
1

前面从学习到使用的过程可以重复很多次，可能在极短的时间内完成，也可能要经过一段相当长的时间。造成这种现象的原因有很多，其中一个比较重要也比较常见的原因就是学习环境的变化。与学生时代的学习方式不同，工作后，我们的学习时间通常都是碎片化的，并且带有很强的主动意识。因此，工作后的学习过程是一个长期坚持的过程，稍有松懈就会导致学习过程的中断，进而导致能力形成的速度放缓。

为了解决好这个问题，我们需要借助于工具，也就是标题所说的高效的个人信息流。所谓信息流，就是对信息的获取、过滤、处理和定期梳理形成知识的过程，如下。

 阅读信息 -> 初级过滤 -> 有用信息 -> 二级过滤 -> 分类 + 标签化 -> 定期梳理 -> 知识
1

在信息流中，对信息的分类和标签化非常重要，直接决定了后续梳理的效率。信息分类是对信息进行类别划分，从基础知识、思考总结和具体使用的维度将类别划分为如下三类：

  • 基础知识：各种基础知识，通常表现为科普贴、知识片段等。
  • 思考总结：对知识成体系地归纳总结。
  • 解决问题：运用知识解决的实际问题记录。

标签是信息的特征，是某一类别内或者多个类别内的多个信息的集合，根据信息内容而定。

能力的背后是知识的支撑，而知识是立体的、分层的。从横向上看，同一层面的知识可以通过信息流快速习得。从纵向上看，上层知识可以在下层知识的基础上通过信息流快速习得。因此，通过信息流可以助力能力快速提升。

根据个人的平时习惯，获取信息的途径可以细分为三类：书籍，PC 端信息，移动端信息。书籍（包括实体书和电子书）本身就是成体系的信息集合，天然具有加速能力形成的条件，因此阅读书籍可以直接进入到信息流的定期梳理环节，非常高效。PC 端信息和移动端信息相对来说往往比较零散，呈点状分布而且容易丢失。比如阅读一篇微信公众号文章或者一篇博客，如果不马上对信息进行梳理沉淀的话，可能就被下一篇文章或者博客的信息冲掉了，形成注意力浪费。因此需要通过信息流对这类信息进行处理和保存。

信息流的载体主要是两部分，一个是对信息的分类和标签化，一个是最终知识的沉淀保存。最终的沉淀可以通过博客的形式保存知识。对信息的分类和标签化需要通过特殊的工具来进行。
前面说到，零散信息主要来自于 PC 端和移动端，因此在 PC 和移动端都需要相应的工具来实现。在 PC 端，我们通常通过浏览器来阅读信息，因此，可以通过浏览器插件的方式，完成信息的收集和过滤。在移动端，由于我们获取信息的途径很多，无法通过单一应用插件的形式完成，并且为了方便将信息过滤后汇总，需要提供一个独立的 App 来完成功能。
在收集完成后，需要将信息进行存储，以待定期梳理。因此需要集中的服务端数据存储。

以上是关于打造高效信息流的初步框架和想法，笔者做了一些功能实现。

信息流只是工具，运用工具可以加快知识到能力的转变速度，但是知识到能力转变的根本还是内在自驱动力。
