# 前端安全之点击劫持（ClickJacking）

点击劫持（ClickJacking）本质上是一种视觉欺骗手段，攻击者使用一个透明的、不可见的 iframe，或者带有欺骗性的图片等覆盖在网页上，然后诱导用户在网页上进行操作，此时用户将在不知情的情况下点击 iframe 页面或者图片。

比如，攻击者可以在一个论坛网站的发布评论按钮出防止一个透明 iframe，iframe 里加载了微博的网页，通过调整 iframe 的大小和位置，攻击者可以做到当用户点击发布按钮的时候，实际上却给某条微博点了赞。

现在很多网站的链接通常以图片的形式出现，比如分享类按钮。攻击者可以通过伪造一个相同的图片，但是链接到钓鱼网站，骗取用户点击，以盗取用户敏感信息。

在移动端，点击劫持通常被称为触屏劫持（TapJacking），攻击原理与 PC 端大致相同。

点击劫持是一种视觉欺骗，通常的攻击手段是 iframe。因此，防御点击劫持的方法也主要从这方面入手。
为了让网站中的 iframe 加载受控的网页，我们可以设置 X-Frame-Options HTTP 头。它有三个取值：

  • DENY
  • SAMEORIGIN
  • ALLOW-FROM origin

当值为 DENY 时，浏览器会决绝当前页面加载任何 frame 页面；若值为 SAMEORIGIN，则 frame 页面的地址只能为同源域名下的页面；若值为 ALLOW-FROM，则可以自定义允许 frame 加载的页面地址。
